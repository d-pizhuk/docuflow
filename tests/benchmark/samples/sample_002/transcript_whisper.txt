In this tutorial I want to show you the full workflow of creating a branch and then opening a pull request on GitHub. This is probably the most common workflow you will use when contributing to a project, whether it's your own or someone else's. So let's start from the repository main page. I already have a repository open here called Demo Project. The first thing we want to do is create a new branch to work on our feature or bug fix. We do not want to work directly on the main branch because that can cause problems for other collaborators. So look at the top left area of the file list, you will see a drop down button that currently says main. Click on that branch drop down button. A small pop over appear with a search field and the list of existing branches. In the search field I'm going to type the name of my new branch. I'll call it Feature Add Login. You will see that GitHub shows a suggestion that says Create branch Feature Add Login from main. Click on that suggestion to create the branch. The page refreshes and the branch drop-down now shows feature ad login, which means we are now on that branch. Great, so now let's make a small change on this branch to simulate doing some work. I will click on the README file to open it. Once the file is open, click the pencil icon in the top right of the file view to edit it. I'll add a new line at the bottom that says Edit login feature documentation. Now scroll down to the commit changes section. You will see a text fill for the commit message. Type something descriptive like add login notes to readme. Make sure the option is set to commit directly to the feature at login branch. Then click the green Commit Changes button. Ok, so now we have a commit on our feature branch. GitHub will often show you a yellow banner at the top of the repository saying that your branch had recent pushes and asking if you want to compare and create a pull request. You can click the Compare and Pull Request button there, or alternatively click on the Pull Request tab at the top. Let's click on Pull Request and then click the green New Pull Request button. On the Pull Request creation page, you need to set the base branch and the compare branch. The base branch is where we want to merge our changes inter, which is main. The Compare branch is the branch with our changes, which is feature add login. Make sure those are set correctly. GitHub will show you a difference of the changes below. You should see the line we added to the readme highlighted in green. Now fill in the pull request title. I'll type add login feature documentation as the title. In the description box, we can add more context about what the pull request does, why you made these changes or link to any related issues. I will add a short note saying this PR adds initial documentation for the login feature. On the right side, you can optionally assign reviewers, add labels or link to a project. For now, I will skip those. When you are ready, click the green button. create pull request button. Your pull request is now open and you can see it in the pull request tab with the title we gave it. Other team members can now review it, leave comments and eventually someone with right access can merge it into main.
