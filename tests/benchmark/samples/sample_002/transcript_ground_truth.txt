In this tutorial I want to show you the full workflow of creating a branch and then opening a pull request on GitHub. This is probably the most common workflow you will use when contributing to a project, whether it's your own or someone else's. So let's start from the repository main page. I already have a repository open here called demo-project. The first thing we want to do is create a new branch to work on our feature or bug fix. We do not want to work directly on the main branch because that can cause problems for other collaborators. So look at the top left area of the file list, you will see a dropdown button that currently says main. Click on that branch dropdown button. A small popover appears with a search field and the list of existing branches. In the search field I'm going to type the name of my new branch, I'll call it feature-add-login. You will see that GitHub shows a suggestion that says Create branch feature-add-login from main. Click on that suggestion to create the branch. The page refreshes and the branch dropdown now shows feature-add-login, which means we are now on that branch. Great, so now let's make a small change on this branch to simulate doing some work. I'll click on the README file to open it. Once the file is open, click the pencil icon in the top right of the file view to edit it. I'll add a new line at the bottom that says Added login feature documentation. Now scroll down to the Commit changes section. You will see a text field for the commit message. Type something descriptive like Add login notes to README. Make sure the option is set to Commit directly to the feature-add-login branch. Then click the green Commit changes button. Okay so now we have a commit on our feature branch. GitHub will often show you a yellow banner at the top of the repository saying that your branch had recent pushes and asking if you want to compare and create a pull request. You can click the Compare and pull request button there, or alternatively click on the Pull requests tab at the top. Let's click on Pull requests and then click the green New pull request button. On the pull request creation page you need to set the base branch and the compare branch. The base branch is where we want to merge our changes into, which is main. The compare branch is the branch with our changes, which is feature-add-login. Make sure those are set correctly. GitHub will show you a diff of the changes below. You should see the line we added to the README highlighted in green. Now fill in the pull request title. I'll type Add login feature documentation as the title. In the description box you can add more context about what the pull request does, why you made these changes, or link to any related issues. I'll add a short note saying This PR adds initial documentation for the login feature. On the right side you can optionally assign reviewers, add labels, or link to a project. For now I'll skip those. When you are ready, click the green Create pull request button. Your pull request is now open and you can see it in the Pull requests tab with the title we gave it. Other team members can now review it, leave comments, and eventually someone with write access can merge it into main.
