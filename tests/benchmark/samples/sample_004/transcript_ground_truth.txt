Alright, in this tutorial I'm going to show you how to set up your Git identity and then make your very first commit. This is one of the first things you should do on any new machine where you install Git, because Git uses your name and email to label every commit you make. So open a terminal. On Windows you can use Command Prompt, PowerShell, Git Bash, or the integrated terminal in VS Code. On Mac or Linux just open your regular terminal application. Once the terminal is open the first thing we will do is configure the global user name. Type git config then two dashes then the word global then user.name and then your name in double quotes. So for example git config --global user.name John Doe. Press Enter to run the command. There is no output if the command succeeds which is normal for Git configuration commands. Now let's set the email address. Type git config --global user.email and then your email address in double quotes. For example git config --global user.email john.doe@example.com. Press Enter. Again no output means it worked. Now let's verify both settings were saved correctly. Type git config --global --list and press Enter. This will print out all your global Git configuration values. You should see user.name equals your name and user.email equals your email address somewhere in that list. Great, now let's actually use these settings by making a first commit. Let's create a new directory for our project. Type mkdir my-project and press Enter. Then navigate into it with cd my-project and press Enter. Now initialize a Git repository in this folder by typing git init and pressing Enter. You should see a message saying Initialized empty Git repository. Let's create a simple file to commit. On Windows type echo Hello World > README.md and press Enter. On Mac or Linux type echo Hello World > README.md and press Enter as well. Now let's check the status of the repository by typing git status and pressing Enter. You will see README.md listed as an untracked file in red. We need to stage it before committing. Type git add README.md and press Enter to add it to the staging area. Run git status again and now you should see README.md listed in green under Changes to be committed. Now let's make the commit. Type git commit -m and then a descriptive message in quotes such as Initial commit and press Enter. Git will output some information including your name and email that it used for the commit, confirming the identity we configured earlier. You will also see the commit hash and the message. Your first commit is now done. You can run git log to see the commit history and verify everything looks correct.
