Alright, in this tutorial I'm going to show you how to set up your git identity and then make your very first commit. This is one of the first things you should do on any new machine where you install git. Because Git uses your name and email to label every commit you make. So open a terminal. On Windows you can use Command Prompt, PowerShell, Git Bash or the integrated terminal in VS Code. On Mac or Linux, just open your regular terminal application Once the terminal is open, the first thing we will do is configure the global username world global then user.name and then your name in double quotes so for example git config double dash global user dot name John Doe press enter to run the command There is no output if the command succeeds which is normal for git configuration commands. Now let's set the email address. email address in double quotes. For example, git config global user.email john.dow at example.com. Press enter. Again, no output means it worked. Now let's verify both settings were saved correctly. Type git config global list and press enter. This will print out all your global git configuration values. You should see user name equals your name and user.email equals your email address somewhere in that list. Great, now let's actually use these settings by making a first commit. Let's create a new directory for our project. mkdir my project and press enter then navigate enter it with cd my project and press enter now initialize a git repository in this folder by typing git init and pressing enter. You should see a message saying initialized empty git repository. Let's create a simple file to commit. On Windows type echo hello world to readme.md and press enter. or Linux type echo hello world to readme.md and press enter as well. Now let's check the status of the repository by typing git status and pressing enter. You will see readme.md listed as an untracked file in red We need to change it before committing type git at readme.md and press enter to add it to the state Run git status again and now you should see readme.md listed in green under changes to be committed Now let's make the commit. Type git commit m and then a descriptive message in quotes, such as initial commit and press enter. Git will output some information including your name and email that it used for the commit, confirming the identity we configured earlier. You will also see the commit hash and the message. Your first commit is now done. You can run git log to see the commit history and verify everything looks correct.
