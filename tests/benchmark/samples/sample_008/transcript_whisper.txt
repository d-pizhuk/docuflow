In this tutorial I will show you how to connect a local git repository to a remote on GitHub and then push your local branch up to it. This is the workflow you use when you started a project locally first and then want to put it on GitHub. So I already have a local git repository initialized on my machine. Let me first verify by typing git status in the terminal and pressing enter. I can see we have some commits already in the log. and we are on the main branch. Now let's go to GitHub to create the remote repository. Open your browser and go to github.com. Create a new empty repository by clicking the plus icon and selecting new repository Give it a name. I'll use my local project name Very importantly, do not check the initialize this repository with a readme option. We want an empty repository because we already have our local history. If you initialize it with a readme, there will be a conflicting... history and your push will fail. Click Create repository. GitHub will now show you a page with setup instructions since the repository is empty. We want the section that says push an existing repository from the command line. You should see the two commands there. The first one is git remote at origin followed by the https or ssh url of your repository. Copy that first command. Switch back to your terminal. Paste and run that git remote at origin command. Press Enter. No output means it worked. Let's verify the remote was added by typing git remote dash v and pressing Enter. You should see origin listed twice. Once for fetch and once for push. both pointing to your GitHub repository URL. Now let's push our local branch to the remote. Type git push dash u origin main and press enter. The U flag sets the upstream tracking relationship, which means in the future you can just type git push without specifying the remote and branch name. you may be prompted to enter your GitHub username and a personal access token as the password. If you haven't set up a personal access token yet, you'll need to create one in GitHub under Settings Developer Settings Personal Access Tokens. Use the token as the password when prompted. Once authentication succeeds, Git will upload your commits and you should see output showing the objects being written and a confirmation that your branch was pushed. Switch back to your browser and refresh the GitHub repository page. You should now see all your files and commit history there. Your local repository is now connected to GitHub and future pushes are just a git push away.
