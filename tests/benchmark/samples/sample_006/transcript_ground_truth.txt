Today I want to walk you through how to create a Python virtual environment and install your project dependencies into it. Virtual environments are really important because they isolate your project's dependencies from other projects and from your system Python installation. This prevents version conflicts and makes your projects reproducible. Okay so open your terminal. First make sure Python is installed by typing python --version or python3 --version and pressing Enter. You should see something like Python 3.11.4 or similar. If you get a command not found error you need to install Python first from python.org. Now navigate to your project folder. Use cd to change to the directory where your project lives. For this example I'll type cd my-project and press Enter. Now let's create the virtual environment. Type python -m venv venv and press Enter. The -m venv part tells Python to run the built-in venv module and the last venv is the name of the folder that will be created to store the environment. You can name it anything but venv is the most common convention and is also in the default gitignore for Python. This command might take a few seconds. When it finishes you will see a new folder called venv in your project directory. Now we need to activate the virtual environment. On Windows type venv\\Scripts\\activate and press Enter. On Mac or Linux type source venv/bin/activate and press Enter. You will know the environment is active because your terminal prompt will change to show the name of the environment in parentheses at the beginning, like (venv) before the usual prompt. Now let's install some dependencies. For this example I'll install the requests library. Type pip install requests and press Enter. You will see pip downloading and installing the package along with any dependencies it needs. Once installation is complete let's also install a second package, I'll install flask as a second example. Type pip install flask and press Enter. Now let's save these dependencies to a requirements file so others can install them too. Type pip freeze > requirements.txt and press Enter. This creates a file called requirements.txt in your project folder with all installed packages and their exact versions. You can verify it by typing cat requirements.txt on Mac or Linux or type requirements.txt on Windows. You should see requests and flask listed along with their dependencies and version numbers. When someone else wants to work on your project they just need to create their own virtual environment, activate it, and then run pip install -r requirements.txt to install everything. To deactivate your virtual environment when you are done just type deactivate and press Enter. The terminal prompt will go back to normal without the environment name prefix.
