In this tutorial I'm going to show you how to write a Dockerfile, build a Docker image from it, and then run your application as a Docker container. Docker is a tool that packages your application together with all its dependencies into a portable container that runs the same way on any machine. So let's assume we have a simple Python Flask application. The project folder contains a file called app.py which is our application and a requirements.txt with the dependencies. The app.py just starts a Flask server that returns Hello from Docker on the home route. The first thing we need to do is create a Dockerfile in the project root. Open your text editor and create a new file called Dockerfile with a capital D and no file extension. The first instruction in a Dockerfile is always FROM which specifies the base image. Type FROM python:3.11-slim on the first line. This tells Docker to use the official Python 3.11 slim image as the starting point. The slim variant is smaller and faster to download than the full image. On the next line type WORKDIR /app. This sets the working directory inside the container to /app. All subsequent commands will run from this directory. Now type COPY requirements.txt . with a dot at the end. The dot means copy into the current working directory which is /app. On the next line type RUN pip install --no-cache-dir -r requirements.txt. This installs the Python dependencies inside the container. We do this before copying the application code because Docker caches each layer and if dependencies don't change this step won't re-run on future builds. Now type COPY . . to copy the rest of the application code into the container. Then type EXPOSE 5000 to tell Docker that the container listens on port 5000. Finally type CMD with square brackets, then the command to start the application. So CMD open square bracket quote python quote comma quote app.py quote close square bracket. Save the Dockerfile. Now open your terminal and navigate to the project folder. Type docker build -t my-flask-app dot and press Enter. The -t flag gives the image a name, in this case my-flask-app, and the dot tells Docker to look for the Dockerfile in the current directory. Docker will now execute each instruction in the Dockerfile and you will see output for each step. This might take a minute the first time because it needs to download the base image. When the build finishes you will see a message saying Successfully built followed by an image ID and then Successfully tagged my-flask-app:latest. Now let's run the container. Type docker run -p 5000:5000 my-flask-app and press Enter. The -p 5000:5000 flag maps port 5000 on your host machine to port 5000 inside the container. You should see the Flask development server starting up. Open your browser and go to http://localhost:5000. You should see the message Hello from Docker in your browser. Your Flask application is now running inside a Docker container. To stop the container press Control C in the terminal.
