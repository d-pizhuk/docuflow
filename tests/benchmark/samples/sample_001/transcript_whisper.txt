Okay, so today I'm going to walk you through the process of creating a brand new repository on GitHub. This is something you will do pretty often when you start a new project, so it's good to know how to do it properly. So, the first thing you need to do is make sure you are actually logged into your GitHub account. If you are not logged in, go ahead and do that now. Just go to github.com and sign in with your credentials. Once you are on the GitHub homepage, you should see your dashboard with your recent repositories and activity feed. Now, in the top right corner of the screen, you will notice a small plus icon next to your profile picture. and click on that plus icon. A small drop-down menu appears and you will see a few options there. We want to click on the one that says new repository. Okay, so now we are on the create a new repository page. The first thing we need to fill in is the repository name. This should be something short, descriptive and ideally lowercase with hyphens instead of spaces. For this example I am going to type hello world as the name. You will notice GitHub gives you a little green checkmark if the name is available, which is nice. Below the name field there is an optional description field. I like to always fill this in because it helps other people and your future self understand what the project is about. So I will type something like my first GitHub repository for practice. Now below that you will see two radio buttons for visibility. You can choose either public, which means anyone on the internet can see. your code or private, which means only you and people you explicitly invite can see it. For this demo I will leave it as public. Scrolling down a little bit you will see a section called initialize this repository. There is a checkbox that says add a readme file. I strongly recommend always checking this box because it creates an initialize. commit and makes the repository easier to clone right away. Go ahead and check that box. You can also optionally add a gitignore template and choose a license here. For now, I will skip those and just focus on the readme. Alright, now scroll all the way down to the bottom of the page and you will see the big green button that says create repository. Go ahead and click that button. GitHub will now create your repository and redirect you to the repository main page. You can see the repository name at the top The readme file is already there with the repository name displayed. And you have the full interface available with the code tab. issues, pull requests, and so on. Your repository is now live and ready to use. You can copy the repository URL from the green code button to clone it locally whenever you are ready.
