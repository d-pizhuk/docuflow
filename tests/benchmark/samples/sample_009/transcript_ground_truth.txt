In this tutorial I want to walk you through how merge conflicts happen in Git and most importantly how to resolve them. Merge conflicts sound scary but once you understand what is happening they are pretty straightforward to deal with. A conflict happens when two branches have made different changes to the same part of the same file and Git doesn't know which version to keep. Let me show you this with a concrete example. I have a repository with a main branch and a feature branch. Both branches have modified the same line in a file called app.py. Let's start on the main branch. Type git checkout main and press Enter to make sure we are on main. Now let's try to merge the feature branch into main. Type git merge feature-branch and press Enter. You will see a message saying CONFLICT and that automatic merge failed, fix conflicts and then commit the result. This is the merge conflict. Don't panic. Let's see which files are conflicted. Type git status and press Enter. You will see app.py listed under both modified and unmerged paths in red. Open that file in your text editor. Inside the file you will see some special markers that Git inserted to show you the conflict. There will be a line with seven less-than symbols followed by HEAD, which marks the beginning of the changes from your current branch which is main. Then the actual content from main. Then a line with seven equals signs which is the separator. Then the content from the feature branch. Then a line with seven greater-than symbols followed by feature-branch which marks the end of the conflict. Your job is to edit this section and decide what the final content should look like. You need to remove all three of those marker lines, the less-than symbols, equals signs, and greater-than symbols. And keep only the content you want, whether that is the main version, the feature version, or a combination of both. In this case I'll keep both changes combined so I'll remove the markers and keep both lines of code. Save the file. Now back in the terminal type git add app.py and press Enter to mark the conflict as resolved. If there were multiple conflicted files you would add each one after resolving them. Now type git status again and you should see the file listed under Changes to be committed in green, meaning it is staged. Now complete the merge by typing git commit and pressing Enter. Git will open a default commit message in your editor saying Merge branch feature-branch. You can keep this message or modify it. Save and close the editor. The merge is now complete. Type git log --oneline to see the merge commit at the top of the history. That's how you resolve a merge conflict. The key steps are: identify the conflicted files, edit them to remove the markers and choose the final content, stage the resolved files, and commit.
