In this video I'm going to show you how to fork a public repository on GitHub and then clone your fork to your local machine. Forking is the standard way to contribute to open source projects. It gives you your own personal copy of a repository that you can freely modify without affecting the original. So let's start by finding the repository we want to fork. For this example, I'll use a simple public repository. Navigate to the repository page on GitHub. You can see the repository name at the top. the file list and various steps like code, issues and pull requests. In the top right area of the repository page, just below the navigation bar, you will see three buttons – Watch, Fork and Star. Click the fork button. GitHub will now show you the create a new fork page. Here you can see the owner drop-down which should default to your own GitHub account. Make sure your account is selected in the owner. a field. You can also change the repository name if you want a different name for your fork. But usually you keep the same name. There is also a checkbox that says copy the main branch only. For most use cases it's fine to leave this checked. but if you need all branches, you can uncheck it. When you are ready, click the green Create Work button at the bottom. GitHub will process this for a second and then redirect you to your fork. You can tell it's your fork because the URL now shows your username in the path and there is a label just below the repository name that says forked and then original repository. Great, so now we have the fork on GitHub. Let's clone it to our local machine. Click the green code button on your fork's repository page. A small drop-down appears with different clone options, HTTPS, SSH, and GitHub CLI. I'll use HTTPS for simplicity. Click the copy icon next to the HTTPS URL to copy it to your clipboard. Now switch to your terminal. Navigate to the directory where you want to store your projects. For example, type cd documents and press enter. Or wherever you kept you keep your code. Now type git clone followed by a space and then paste the URL you just copied. Press enter to run the command. Git will connect to GitHub and download all the repository files. You will see progress output showing the objects being received. Once it finishes, type cd followed by the repository name and press Enter to navigate into the cloned folder. type ls on Mac or Linux or dir on Windows to list the files and confirm they are all there. You now have a local copy of your fork. You can make changes, commit them, push them to your fork on GitHub, and then open a pull request to the original repository.
